
Summary:
On a benchmark test, creating capture events in bulk takes ~50% of the time it takes to create capture events individually
In a real-life scenario, creating capture events in bulk takes ~20% of the time it takes to create capture events individually

Creating events individually: using arena and "fixed64"
Creating events in bulk: using arena and "repeated fixed64"

Note: this is the creation of the capture events alone, there is possibly more overhead later down the line
      for sending lots of individual events vs bulking them.

1. Benchmark
On gamelet (orbit/src/OrbitProducer/LockFreeBufferCaptureEventProducerTest.cpp)
./OrbitProducerTests --gtest_filter=ApiEvent*
